EXPOSED

The EXPOSED database is a novel, comprehensive and exclusive database on environmental exposure to nanoparticles designed by the LabEx SERENADE.
Based on the Safe-by-Design approach, the EXPOSED database can be used as benchmarking source to improve both safety and efficacy of manufactured nanomaterials.
EXPOSED is available as downloadable SQL file for free under an Open Database License v1.0 (GUI).
A JAVA Graphical User Interface is freely available to provide direct managing, searching and browsing.
The web portal of the EXPOSED database was built in HTML, CSS, JavaScript and JQuery to make it more attractive.

About

The EXPOSED database - for Environmental eXPOsure to nanomaterials Database - is a computational infrastructure for toxicological data management of engineered nanomaterials.

It has developed an environmental exposure database (aquatic mesocosms data, release from products data, ecotox data, â€¦) based on results obtained within the SERENADE's team.

EXPOSED aims to predict and explain the nanomaterial behavior in different systems would be investigated as well as the potential risks with nanomaterials on the environment, human health or the end of life of nanoproducts.

In summary, EXPOSED is a distinct database resource providing 6336 manually involved in 41 different experiences about different types of nananoparticles, mainly the Ag, CeO2, Cu, Ce and so on.

Prerequisites

You need a MySQL database server (5.0+) and run the commands below through a user that has the following privileges:


SELECT, INSERT, UPDATE, DELETE, 
CREATE, DROP, RELOAD, REFERENCES, 
INDEX, ALTER, SHOW DATABASES, 
CREATE TEMPORARY TABLES, 
LOCK TABLES, EXECUTE, CREATE VIEW

See usage in the MySQL docs.

Installation:

  1. Download the repository
  2. Change directory to the repository

Then run


mysql < employees.sql

If you want to install with two large partitioned tables, run


mysql < employees_partitioned.sql

Testing the installation

After installing, you can run one of the following


mysql -t < test_employees_md5.sql
# OR
mysql -t < test_employees_sha.sql
